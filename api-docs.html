接口文档 · 轻量集成说明

在同一套服务中调用 /check 与 /tag

本页聚焦现有 HTTP 接口的实际调用方式，覆盖文件上传与服务器本地路径两种模式、可选 Bearer Token 认证，以及代表性的返回结构，方便直接接入审核或标签流程。

POST/check 内容审核
POST/tag 自动标签
AUTH配置后使用 Bearer Token
Quick Summary

接入前先确认这三件事

两个接口都使用 multipart/form-data。你可以上传真实文件，也可以提交服务器本地路径；如果服务端配置了 token，请在请求头中附带 Authorization: Bearer <token>。

Request Modes
file 或 path
Auth Header
Authorization: Bearer <token>
Content Type
multipart/form-data
Response Shape
status + filename + result

通用请求约定

这两个接口共享相同的请求风格。前端首页使用 file 上传模式，API 也支持直接传入服务端可访问的 path 字段，用于处理已在服务器上的文件。

支持的提交模式

  • file：使用 multipart/form-data 上传文件内容。
  • path：提交服务器本地文件路径，不上传二进制内容。

鉴权行为

当服务端配置了认证 token 时，请附带 Authorization 请求头，格式为 Bearer <token>。未配置时可省略该请求头。

请求头示例

Authorization: Bearer <token>

/check · 内容审核

返回 normal 与 nsfw 分数，并根据阈值给出最终判定。适合上传前审查、自动化审核与安全开关场景。

POST /check

表单字段

  • file：上传待检测文件。
  • path：服务器本地文件路径。

代表性返回字段

  • status 成功时为 success。
  • filename 返回用于分析的文件名。
  • result.normal / result.nsfw 对应 normal 与 nsfw 分数。

使用 file 模式

curl -X POST \
  -H "Authorization: Bearer <token>" \
  -F "file=@/path/to/image.jpg" \
  http://localhost:3333/check

使用 path 模式

curl -X POST \
  -H "Authorization: Bearer <token>" \
  -F "path=/path/to/image.jpg" \
  http://localhost:3333/check

响应示例

{
  "status": "success",
  "filename": "sample.jpg",
  "result": {
    "normal": 0.9731,
    "nsfw": 0.0269
  }
}

/tag · 自动标签

为图片或视频返回标签列表，并在可用时附带文件类型与视频分析帧数，适合内容分类、检索增强与素材整理流程。

POST /tag

表单字段

  • file：上传图片或视频文件。
  • path：服务器本地图片或视频路径。

代表性返回字段

  • result.type 返回文件类型，例如 image 或 video。
  • result.tags 标签数组，包含 key、label、score。
  • result.frames_analyzed 视频场景下可返回参与聚合的帧数。

使用 file 模式

curl -X POST \
  -H "Authorization: Bearer <token>" \
  -F "file=@/path/to/video.mp4" \
  http://localhost:3333/tag

使用 path 模式

curl -X POST \
  -H "Authorization: Bearer <token>" \
  -F "path=/path/to/video.mp4" \
  http://localhost:3333/tag

响应示例

{
  "status": "success",
  "filename": "sample.mp4",
  "result": {
    "type": "video",
    "frames_analyzed": 12,
    "tags": [
      {"key": "anime", "label": "动漫", "score": 0.9123},
      {"key": "girl", "label": "少女", "score": 0.8642},
      {"key": "portrait", "label": "人像", "score": 0.6211}
    ]
  }
}

集成说明

这些补充说明与当前页面行为保持一致，可帮助在前端或服务端封装时减少歧义。

上传模式

首页上传器当前使用 file 模式，并依据所选功能自动切换到 /check 或 /tag。

路径模式

path 适合文件已在服务器本地存在的场景，调用方只需要提交表单字段即可。

认证模式

Bearer Token 仅在服务端配置后才需要；未启用认证时，请求可不带 Authorization 头。
